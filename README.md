# 🎤 PoketMic — スマホ ワイヤレスマイクシステム

スマートフォンをワイヤレスマイクに変え、会場スピーカーへリアルタイム音声出力するローカルネットワーク対応システム。

---

## 必要環境

- Node.js v16 以上（会場PCにインストール済みであること）
- 全員が同じ WiFi に接続していること
- 参加者スマホ：iOS Safari / Android Chrome（マイク許可が必要）
- 会場PC：Chrome / Edge 推奨

---

## セットアップ手順（初回のみ）

```bash
# 1. フォルダに移動
cd poketmic

# 2. 依存パッケージをインストール
npm install
```

---

## 毎回の起動手順

```bash
node server.js
```

起動すると以下が表示されます：

```
========================================
  🎤 PoketMic サーバー起動！
========================================
  管理者画面   : https://192.168.1.10:3443/admin.html
  出力画面     : https://192.168.1.10:3443/output.html
  参加者URL    : https://192.168.1.10:3443/mic.html
  ルームコード : ABC123
========================================
```

## 当日の流れ

| 担当 | 操作 |
|------|------|
| オペレーター | `node server.js` を起動 |
| 会場PC | `/output.html` を開き「▶ 出力を開始」をクリック、PCをPAに接続 |
| オペレーター | `/admin.html` でQRコードを画面表示 |
| 参加者 | スマホでQRコードを読み取り → ルームコード入力 → 接続 |
| 参加者 | ボタンを押している間だけ話す（Push-to-Talk） |

---

## スマホで「セキュリティ警告」が出る場合

ローカル HTTPS の自己署名証明書のためです。以下の手順で通過してください。

**Android Chrome:**
「詳細設定」→「〇〇.〇〇.〇〇.〇〇 にアクセスする（安全でない）」

**iPhone Safari:**
「詳細を表示」→「このWebサイトを閲覧」

---

## ハウリング防止の仕組み

| 対策 | 内容 |
|------|------|
| 送受信完全分離 | スマホは**送信専用**でスピーカーの音を受信しない |
| Push-to-Talk | ボタンを押した時だけ送信・待機中はゼロ |
| ブラウザ AEC | `echoCancellation: true` でエコーキャンセル |
| サーバー soft clip | `tanh()` で複数人ミックス時の音割れを防止 |
| バッファ再生 | 出力側で 150ms バッファリングしてジッター吸収 |
